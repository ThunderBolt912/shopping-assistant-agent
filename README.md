### 3️⃣ Discount‑code redemption tool (the new feature)
```python
DISCOUNT_CODES = {"WELCOME50": 50, "SUMMER20": 20}
redeemed_codes: set[tuple[str, str]] = set()

def redeem_discount_code(user_id: str, code: str) -> str:
    """Redeem a single‑use discount code.
    * Validates the code exists.
    * Guarantees a (user_id, code) pair can be used only once.
    * Returns a friendly success/failure message.
    """
    normalized = code.upper().strip()
    if normalized not in DISCOUNT_CODES:
        return f"❌ Code '{code}' is invalid."
    key = (user_id, normalized)
    if key in redeemed_codes:
        return f"⚠️ Code '{code}' has already been redeemed by this user."
    redeemed_codes.add(key)
    discount = DISCOUNT_CODES[normalized]
    return f"✅ Success! You've received a {discount}% discount."
```
- **Stateful**: the `redeemed_codes` set lives in memory for the lifetime of the process, enforcing one‑time use.
- **Exposed as a tool**: the function is added to the agent’s `tools` list, so LLM can call it when the user asks *“Apply my discount code WELCOME50”*.

### 5️⃣ Root **Workflow** – the App entry point
```python

    root_agent=root_agent,

```
- `App` wraps the `root_agent` and exposes a HTTP endpoint (via the ADK runtime) that the playground or any client can call.
- When a request arrives, the ADK runtime:
  1. Sends the user prompt to the Gemini model.
  2. If the model decides a tool is needed, it calls the appropriate Python function.
  3. The tool’s return value is fed back into the conversation, allowing the LLM to continue with the new information.
- This **root workflow** is therefore the complete request‑response loop for the shopping assistant.

## How the Discount Flow works in practice
1. **User**: *“I have the code WELCOME50, can I use it?”*
2. **Gemini** decides it needs to call `redeem_discount_code`.
3. **ADK runtime** executes the Python function with the supplied `user_id` (usually extracted from the request metadata) and the code.
4. The function checks `DISCOUNT_CODES`, validates one‑time use, updates `redeemed_codes`, and returns a success string.
5. The success message is sent back to the user as part of the LLM’s final answer.
